分支操作[返回]

要理解分支，请参考《Pro Git》

git的分支与svn的不一样，这点很重要

罗列分支

$git branch

列出所有分支，带点的是当前分支

$git branch -v

列出所有分支，同时列出最后一次提交

$git branch --merged

列出已经与当前分支合并了的分支

$git branch --no-merged

列出没有与当前分支合并了的分支

新建分支

$git branch <branch_name> [commit_obj]

可以从某次提交结果开始建立分支，默认是最近一次提交

切换分支

$git checkout <branch_name>

新建分支之后不会立刻切换到新建分支，如果要立刻切换，如下：

$git checkout -b <branch_name> [commit_obj]

删除分支

$git branch -d <branch_name>

合并

$git merge <branch_name>

把制定分支的内容合并到当前分支

合并的原理也要理解，参考《Pro Git》

每个分支代表一种更新，因此在相同位置的两种更新会发生冲突

发生冲突的合并不会提交，要等到人为解决冲突之后才能合并
