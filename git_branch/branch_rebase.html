衍合[返回]

注意衍合与合并的区别

衍合的特点：与合并的结果是一样的，但是能使提交历史更加整洁

更加整洁的提交历史可以方便管理者维护代码

因此如果只是个人项目或者小项目，没必要使用衍合

衍合

$git rebase <branch-name>

把当前分支衍合到指定分支

注意与合并指令的区别：合并是把指定分支合并到当前

两条指令虽然结果一样，但是执行指令之前所在分支不一样

要切换到正确的分支执行指令

助记：两条指令的指定分支在指令执行结束之后都不会改变

$git rebase <branch1> <branch2>

把branch2与branch1的差异衍合到branch1中

branch1不变

通常为$git rebase 主分支 特性分支

$git rebase --onto <branch1> <branch2> <branch3>

把branch2与branch3的差异衍合到branch1中

branch1与branch2不变
