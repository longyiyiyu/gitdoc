远程分支[返回]

远程分支是对远程仓库状态的索引

它们是一些无法移动的本地分支，只有与远程仓库同步才会移动

它们就像书签，记录上次同步远程仓库时各分支的位置

用(远程仓库名)/(分支名)表示远程分支

推送远程分支

$git push <remote-name> <branch-name>[:<remote-branch-name>]

如果省略远程分支名，则创建同本地分支同名的分支

跟踪分支

从远程分支检出的本地分支称为跟踪分支

它和远程分支有直接关系，在跟踪分支上执行git push或者git pull会自动同步分支的数据

$git checkout --track <remote-name>/<remote-branch-name>

$git checkout <remote-branch-name>

上面两条指令是一样的，都会在本地创建一个跟踪分支，并与远程分支同名

$git checkout -b <branch-name> <remote-name>/<remote-branch-name>

创建名为branch-name的本地分支，跟踪远程分支

删除远程分支

$git push <remote-name> :<remote-branch-name>
