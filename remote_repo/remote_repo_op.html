使用远程仓库[返回]

罗列远程仓库

$git remote

列出所有配置的远程仓库，其中origin是当初的克隆源仓库

加上-v选项可以查看远程仓库的url

添加远程仓库

$git remote add <shortname> <url>

添加远程仓库的意义仅仅是添加一个助记的url

然后就可以用shortname替代url使用

从远程仓库抓取数据

$git fetch <remote-name>

remote-name用于区别

上述指令只把本地仓库没有的数据抓取到本地

抓取下来之后，不会自动合并，如果要自动合并，如下：

$git pull <remote-name>

推送数据到远程仓库

$git push <remote-name> <branch-name>

把本地的branch-name分支推送到远程仓库remote-name

查看远程仓库的信息

$git remote show <remote-name>

查看远程仓库的信息，包括：

  • 克隆地址
  • 处于跟踪状态的分支
  • 未跟踪的分支，即还没有同步到本地仓库的分支
  • 已经同步到本地，但远程仓库已经删除了的分支
  • git push缺省推送的分支
  • git pull缺省合并的分支

重命名远程仓库

$git remote rename <remote-name> <new-name>

删除远程仓库

$git remote rm <remote-name>
