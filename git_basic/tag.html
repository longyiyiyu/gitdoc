标签与提交[返回]

标签是帮助标记位置和添加助记的“东西”

git标签一般是标记发布版本号和说明的

标签是打在某个提交上的，因此标签可以看做是某次提交的alias

罗列标签

$git tag

列出所有标签

$git tag -l "v1.4.*"

列出所有符合条件的标签

新建标签

$git <tag_name>

新建一个轻量级标签，轻量级标签只有名称

$git -tag -a <tag_name> -m "<message>"

新建一个含附注信息的标签

$git -tag -a <tag_name> <commit_object> -m "<message>"

给某个提交打标签

注意：git中是用校验和字符串指代提交对象的

分享标签

默认git push不会把标签推送到远端服务器

$git push <branch_name> <tag_name>

$git push <branch_name> --tags

推送所有标签

标签与提交的常用操作

注意：下面所有操作标签和提交对象可以替换使用

$git diff <tag_name> <commit_obj>

比较两次提交的差异

提示：HEAD指代最近一次提交

$git branch <branch_name> <tag_name>

以某次提交为基底新建分支

要在某个版本上开始工作，可以使用版本标签来新建分支

$git grep <grep_string> <tag_name>

对某次提交之后的所有文件内容进行grep搜索

$git show <tag_name>:<file>

查看某次提交的指定文件的内容
