提交更新到仓库[返回]

有了git仓库之后，就可以管理项目内的所有文件，just you want!!

理解git仓库文件的各种状态

  • 未跟踪

    • 不受git系统管理的文件，通常为临时文件和日志文件

    可以使用.gitignore文件指定git忽略的文件

  • 未更新

    • 自上次提交之后没被更新过的文件。记住，版本控制是控制变更，即只关注被更改的东西

  • 已更新

    • 自上次提交之后被更新过的文件

  • 已暂存

    • 放入暂存区的文件。重点理解暂存区，一般git的控制是对暂存区的控制

    重点记住：每次提交更新都只提交暂存区内的文件

    每次提交之后，暂存区都会被清空等待下次提交

    每次提交，都不会记录已更新文件，除非被放入暂存区

检查文件状态

$git status

跟踪新文件

$git add <files>

暂存已修改文件

$git add <files>

跟踪新文件也是把新文件放入暂存区，所以使用同一条指令

查看暂存区域内的文件与当前工作目录下的文件的差异(更新)

$git diff

也就是修改之后还没有暂存起来的内容

查看暂存区域内的文件与上一次提交文件的差异(更新)

$git diff --staged

提交更新

$git commit

自动打开默认编辑器，输入一行提交说明之后提交

$git commit -m "<message>"

直接提交

跳过使用暂存区

$git commit -a

使用暂存区可以有选择的提交更新，有时候为了简单，可以直接跳过暂存区，提交所有更新过的文件

移除文件

$git rm

同时删除文件

移除跟踪文件

$git rm cached

不删除文件，只是不跟踪文件

移动文件

$git mv <file_from> <file_to>
